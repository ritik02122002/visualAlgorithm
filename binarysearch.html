BINARY SEARCH

Binary Search is a searching algorithm for finding an element's position in a sorted array by repeatedly dividing the search interval in half.

Problem Statement:

Given a sorted array "arr" of integers and an element "key". We have to search if "key" exists in "arr" or not using binary search(Assuming array indexing starts from 0)

Pseuodocode:

Binary Search ( Array arr, Element key)

Step 1: Let start = 0 and end = n-1.

Step 2: Compute mid as the average of start and end, rounded down (so that it is an integer).

Step 3: If arr[mid] equals key, then stop. You found it! Return mid.

Step 4: If the mid was too low, that is, arr[mid] < key, then set start = mid + 1.

Step 5: Otherwise, the mid was too high. Set end = mid - 1.

Step 6: Go back to step 2.

Complexity Analysis:

Time Complexity:

Best Case: Best case is when the element is at the middle index of the array. It takes only one comparison to find the key element. So the best case complexity is O(1).

Worst Case: In the worst case, the key might be present at the last index i.e., opposite to the end from which the search has started in the array. So the worst-case complexity is O(N) where N is the size of the array.

Average Case: O(N)

Space Complexity:

O(1) as except for the variable to iterate through the array, no other variable is used.
