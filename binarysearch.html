BINARY SEARCH

Binary Search is a searching algorithm for finding an element's position in a sorted array by repeatedly dividing the search interval in half.

Problem Statement:

Given a sorted array "arr" of integers of size N and an element "key". We have to search if "key" exists in "arr" or not using binary search(Assuming array indexing starts from 0).

Pseuodocode:

            Binary Search (Array arr, Element key, Size N)
            
                Step 1: Let start = 0 and end = N-1.
                Step 2: Compute mid as the average of start and end, rounded down (so that it is an integer).
                Step 3: If arr[mid] equals key, then stop. You found it! Return mid.
                Step 4: If the mid was too low, that is, arr[mid] < key, then set start = mid + 1.
                Step 5: Otherwise, the mid was too high. Set end = mid - 1.
                Step 6: If start > end, return key not found.
                Step 7: Go back to step 2

Complexity Analysis:

Time Complexity:

Best Case: Best case is when the element is at the middle index of the array. It takes only one comparison to find the key element. So the best case complexity is O(1).

Worst Case: The worst-case scenario of Binary Search occurs when the target element is the smallest element or the largest element of the sorted array. In each iteration, the search gets reduced to half of the array. So for an array of size N, there are atmost logN iterations. Therefore, the Worst Case Time Complexity of Binary Search is O(logN).

Average Case: The average case arises when the target element is present in some location other than the central index or extremities. The time complexity depends on the number of comparisons to reach the desired element. Therefore, the overall Average Case Time Complexity of Binary Search is O(logN).

Space Complexity:

O(1) as no extra space is used
