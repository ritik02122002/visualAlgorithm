LINEAR SEARCH

Linear Search is defined as a sequential search algorithm that starts at one end and goes through each element of an array until the desired element is found, otherwise the search continues till the end of the data set

Problem Statement:

Given an array "arr" of integers and an element "key". We have to search if "key" exists in "arr" or not(Assuming array indexing starts from 0)

Pseuodocode:

Linear Search ( Array arr, Element key)

Step 1: Set i to 1

Step 2: if i > n then go to step 7

Step 3: if arr[i] = key then go to step 6

Step 4: Set i to i + 1

Step 5: Go to Step 2

Step 6: Print key Found at index i and go to step 8

Step 7: Print key not found

Step 8: Exit

Complexity Analysis:

Time Complexity:

Best Case: In the best case, the key might be present at the index 0. So the best case complexity is O(1)

Worst Case: In the worst case, the key might be present at the last index i.e., opposite to the end from which the search has started in the array. So the worst-case complexity is O(N) where N is the size of the array.

Average Case: O(N)

Space Complexity:

O(1) as except for the variable to iterate through the array, no other variable is used.
